    修課心得
    台大, 心得

    這學期的一個轉變大概是雙輔日文系和申請日本交換無望了吧，我也沒有打算要再申請了。
    之前為了這些會特別注意 GPA，但這學期開始就覺得算了，對於選課的考量變成「付出和收獲成不成比例」。
    一方面是想修的課已經修的差不多了，一方面是這學期開始覺得時間很珍貴，希望自己可以花大部分時間在興趣上。
    所以會希望我修了一堂課，除了學分以外還可以得到其他收獲（樂趣 / 知識 / 人際 / 錢 等等）。

    之前為了顧 GPA ，會讓自己修課時保有餘裕，但既然現在不需要了，這學期就想看看自己的極限在哪裡，所以就試著完全不考慮後果地安排一堆事情。
    原本加退選的時候有成功拿到 31 學分，但後來 NTU COOL 實習上了之後就只好退了一些。
    總之最後總共 22 學分 + NASA + NTU COOL 實習 + ADA 助教 + Make Reality Space 管理員。

    結果來說，差不多這就是我的極限了吧，雖然客觀來講不是到真的沒有時間，但這一學期下來確實心態上很累，到第 10 週的時候已經是一個完全不想做事的厭倦狀態了。

    當初原本還在猶豫要不要接 NASA 一階助教，還好沒接。
    但其實我覺得當 NASA 助教看起來很好玩的說，下次有機會可能會接。

    以下是慣例的定義。

    甜度：認真修課的請況下，獲得分數的高低。

    1：有及格就不錯了
    2：不至於被當，但分數不好看
    3：分數中規中矩，A+ 不太可能
    4：就算沒有 A+ 也不會拿到太差的分數
    5：認真的話 A+ 沒什麼問題

    課後時間：課後需要的時間。

    收獲：在這堂課學到的知識量。

    1：基本上沒有
    2：雖然有但不多
    3：有一些知識
    4：有一定程度的知識量
    5：滿滿的收獲，物超所值

    綜合推薦度：是否會推薦別人來修這堂課。

    1：逃げろ!
    2：非修不可再修
    3：如果缺課還是可以
    4：時間允許建議修
    5：有時間必修

    必修課程

    計算機結構 - 洪士灝教授（3 學分）

    • 分類：必修
    • 成績：B
    • 甜度：3
      • 但看大家的分數好像蠻高的，沒想到我居然只有 B
    • 收獲：3
    • 推薦度：2
      • 除了必修以外好像沒有修的理由
    • 課後時間：
      • 作業：1 小時 / 份，因為我都丟給 AI 寫
      • 期中：10~15 小時
      • 期末：3~4 小時

    評分方式

    五次作業共 30%、期中 30%、期末 40%。

    上課方式

    投影片是教科書附的而不是教授做的，再加上教授感覺沒有備課，上課常常照著投影片唸，有時也會上到一半看不懂投影片上的內容而停頓，又或是上到一半發現投影片寫錯。
    不過聽說去年也有這個狀況，我猜教授本人也沒有要修正的意思。

    原本聽說這個教授段考會考上課閒聊時講過的小知識，所以姑且前半學期還是有去聽課，但後來發現我去上課還是只是在做自己的事，加上期中大暴死，所以後半學期就乾脆不去了。

    雖然可能只是我對這一科提不起興趣而已。

    作業

    大致上就是上課的基本概念。
    我都直接丟給 AI，然後叫它附上解題思路。這樣子我就可以在完成作業的同時順便複習。
    啊 verilog 的部分我直接完全交給 AI。

    每次作業都會有一大題叫你用 AI 寫，有點意義不明。
    之前有一題要我們叫 AI 預測 code 執行時間，並實際執行 code 來判斷 AI 猜得準不準，結果那份 code 好像需要跑 24 小時以上，害我工作站帳號被鎖。

    期中期末

    大致上分成名詞解釋題、長文閱讀、應用題。
    出題單元極度不平均，今年和去年都發生題目集中某一個單元，其他單元幾乎或完全沒有出題的狀況。

    期中的時候還蠻認真唸的，我自認有把段考範圍唸好，但還是收了個爛成績。
    去年的名詞解釋題好像只有 4 題，所以我當初打算大致記一下重要名詞就好，但今年考了 10 題，而且還剛好全部不在我的大抄上。
    原本聽說長文閱讀題可以當成學測英文從題目中找答案，結果今年的題目在題目裡沒有答案。
    還有我以為會考手寫組合語言和機器語言，所以把大抄額度幾乎拿去寫語法規則，結果完全沒考。
    應用題雖然沒什麼錯，但前兩大題蠻慘的。

    期末考就乾脆隨便唸了，讀了一遍 gemini 做的重點整理 + 一遍原本的 PPT 而已，大抄基本上是叫 gemini 把所有 PPT 上出現的專有名詞附上解釋，再加上重點整理，然後用超小字體全部擠進去一張 A4 紙裡面。
    進考場的時候我覺得我根本什麼都不會，我甚至有時候連題目在講什麼都看不懂，幾乎是看到題目才現場讀重點整理去理解題目的意思。
    但必須講這次考試策略蠻成功的，沒記錯的話所有名詞解釋題都有在大抄上，應用題的公式和關鍵概念也有抓到，長文閱讀還是一樣大爆死就是了。
    總之用了大約三分之一的讀書時間換來了和期中差不多的成績，還可以接受吧。

    感想

    由於這學期決定不捲 GPA 了，所以對這科的態度很隨便，基本上是抱著用最少時間打發這一科的打算。

    話說我學期間還在抱怨說我這輩子大概不會再碰到計結的知識了，結果帶日本高中生旅遊的時候突然被問半加法器的原理，馬上遭報應，笑死。
    不過我確實不想再碰計結就是了。

    選修課程

    前瞻資訊科技一 - 蔡欣穆教授（3 學分）

    • 分類：系上選修
    • 成績：A
    • 甜度：4
    • 收獲：2
    • 推薦度：4
    • 課後時間：大部分的作業 1~2 小時左右就可以完成

    評分方式

    每個教授給 0~100 分，全部教授取平均。

    上課方式

    基本上是每個教授講他們的研究內容相關的知識，以及介紹他們的專題或研究是在做什麼。

    黎士瑋

    實驗室主要在研究作業系統，提到他們實驗室會經常需要讀 kernel code。

    上課主要是講 SP 和 OS 的東西，以及再更進階一點的內容。

    作業是給你的一個小型作業系統，按照開機流程成功開機就有分，修改開機參數來加速開機速度的話就可以拿滿分。
    這份作業最麻煩的反而是跑 code 本身，不知道為什麼我的工作站 docker 跑不起來，本機的 windows docker 也一直報錯，後來在工作站上開了一個 ubuntu 虛擬機才成功。

    蔡欣穆

    實驗室是在做可見光通訊，也就是利用可見光波段而不是平常 wifi 的頻率來進行通訊。因此這種通訊方式可以結合照明設備，達到極高的覆蓋率和定位精準度。

    上課方式我覺得是這學期所有教授之中數一數二好懂的，課程偏向介紹科技而不是講理論本身，主要是講解可見光通訊是什麼，以及它的實際應用。
    除了電磁學的部分都聽得懂，我很抱歉我物理沒學好。

    作業最後沒有出，不確定是不是因為教授太忙的關系。

    另外教授也提到他的實驗室偏向實務導向，也會有比較多的企業合作，以前聽說還可以練習開飛機，羨慕了。
    如果我是大二修到這門課的話可能大三專題就會去蔡欣穆教授的實驗室吧。

    陳維超

    這個教授好像沒有開課，平常也在企業活動居多，我是第一次聽到這個教授。

    第一堂課講的是關於 GPU 的時代發展以及 GPU programming，講了一些控制 GPU 底層運作的語法。
    第二堂課講了加密運算，探討資料加密後是否能夠保持四則運算的交換律、結合律等等性質，這樣我們就可以直接對加密後的資料運算而不用先解密完運算後再加密，這樣一來對效率和安全性都有幫助。

    作業有兩份，第一堂課課後出了一些 GPU 運作以及企業導向的問題，在第二堂課時展示同學的回答。
    第二堂課結束後要我們用 python 實作加密後運算，並把運算結果交回來，整體的程式邏輯已經寫在作業要求了，只要翻譯成程式語法就好。

    羅紹元

    這堂課在講如何用一些小技巧讓 AI 答錯，以及如何防止 AI 被這些小技巧干擾。

    整體上蠻好懂的，這堂課用了比較 high level 的角度去說明有哪些常見的破壞方法、這些方法需要有哪些成立條件、有哪些防禦方法、它們的代價和優缺點又是什麼。

    沒有作業，而是在第二堂課結束前有小考，考這兩堂課的上課內容以及教授的個人資訊。
    從後來調分開根號乘十來看，應該大家是都考蠻慘的。
    話說我有猜到會考教授博士班的學校校名，但結果考試時還是忘了拼法。

    鄧善元

    上課部分基本上和互動裝置自造與發明的前幾堂課差不多，主要是在介紹人和機器的互動，想了解更多的話可以看下面那一篇互動裝置自造與發明的心得。

    這門課主要在探討怎麼樣在虛擬空間產生觸覺回饋，然後不可避免地會講到關於人體神經和肌肉等等比較生物學層面的內容。
    不過這門課沒有太多著墨在生物層面，大致上是直接告訴我們人體有這個機制，比較多的時間是在問題發想和結果展示

    作業是要我們找一個觸覺體驗並找兩篇論文來說明這個觸覺體驗要如何實現。

    張原豪

    第一堂上課好像是在講記憶體優化，如何讓記憶體可以發揮超過 100% 的使用率。
    第二堂偏向計結，在講 SSD 的存取機制，但我沒有聽懂所以不太清楚具體內容。
    作業是回答幾個問題，以及寫下自己在上課中學到什麼。

    感想

    這堂課可以讓學生大致了解每個教授的研究領域，這樣末來選專題的時候就會有一些方向。
    其實我大一大二就有想要修這堂課，但剛好每次都衝堂沒辦法修，這學期修下來也發現了不少我感興趣的研究內容，有點相見恨晚吧。

    互動裝置自造與發明 - 鄧善元教授（3 學分）

    既然這堂課叫做互動裝置自造與發明，那就請大家實際發明一個裝置吧

    • 分類：系上選修
    • 成績：A
    • 甜度：4.5
    • 收獲：4.5
    • 推薦度：5
    • 課後時間：
      • 每週作業：3~4 小時
      • 期中 / 期末 project：10~20 小時

    評分方式

    出席 10%，五次個人作業 + 小組的期中期末 project（比例不確定）。

    跟據期初第一堂課，給分標準如下：

    • Finish hands-on in class: B
    • Fulfill requirements: A
      • systems, reports, videos, demo
    • BONUS: A+

    但我作業搞砸好幾次，期末 project 還被說太簡單，這樣還有 A 我猜應該不少人有 A+ 吧。

    上課方式

    這位是這學期新來的教授，這個教授上課時非常的有抑揚頓挫，對於習慣了資工系平板時上課的人而言有點驚訝。
    上課我個人覺得蠻有趣的，會由淺入深的進入這堂課的主題，節奏也控制得很剛好，不會短時間塞一堆知識，上課也幾乎不會有重覆性 / 不重要的內容，而且每次都可以在剛好 3 小時的時間把課程講完。
    通常新來的教授常常會不小心講太難或太快，但這個教授的課完全沒有這個問題，而且也確實地把作業會用到的概念和工具教完而不是丟給學生自己摸。
    大推。

    每次上課會介紹一種新的裝置原理，前半學期著重在機械結構，後半學期則是電路相關。
    第二堂課教了 living hinge，由助教教學 2D 建模，然後作業則是要我們設計一個 living hinge 結構並用雷切機切出來。
    後面也有像是 3D 建模和 3D 列印機、然後讓我們自己設計一個舒壓小物等等的內容。

    期中 project 是三人一組，隨機抽一個物品，然後要我們改造它。
    我們這一組抽的是手錶，我負責的是雙層按鈕，另外兩個人做了搖桿和轉盤。

    下半學期開始教電路裝置，但我這部分其實沒有很懂上課的內容，不太理解那些電容電感的互動、arduino 的操作、又或是那些電路裝置為什麼要這樣接等等。
    這半學期我甚至連作業都做不太出來，只能照著上課投影片接一次看會不會動，bouns 那些基本上是做不來了。

    失敗的喇叭


    最後期末 project 一樣是三人一組，要我們自己發想一個裝置，主題不限。
    為了和期末週錯開，教授把 deadline 提前到第 15 週，結果變成我要一邊改 ADA 考卷一邊處理期末。

    還有「ありがたい＋ございます⭢ありがたうございます⭢ありがとうございます」本質上也是一樣的文法，但省略了中間的變型的話就會讓人覺得文法很繁雜。
    很多我覺得本身屬於同一種文法的東西都被拆解成了好幾個 edge case，反而我覺得理解上有點混亂。

    另外一個我覺得有點多餘的是舊國名吧，雖然舊國名在古日文裡面確實有不小的重要性，但我覺得至少文法和常用單字還是更重要一些。
    花草的部分我也覺得有點太早，雖然確實是古典文學沒錯，但文法教到一半突然開始講詩還是覺得有點突兀。

    期中期末

    段考的題型其實在我預料之外，我好像只答對文法題吧。
    第一大題是給古文寫法，要我們寫出讀音和漢字，出題單字來源是那張極度複雜的連音表，但我根本沒有看那張表所以我只有寫讀音，漢字的部分我幾乎沒答出來。
    然後舊國名的考法，我一開始以為是給現在地名，要我們寫舊國名。（神奈川⭤相模之類）
    結果實際考法是給一個區域，要我們自己挑三個國家寫出舊國名和現在的國名。
    這大題暴死。

    期末好一點，一方面是因為古文助動詞我本來就比較熟，另一方面是上課用的文章我或多或少都有一些印象，不過不確定最終成績是多少就是了。

    感想

    原本會想修這堂課是希望自己可以學會讀古日文，雖然我以前有讀過古典文法，但一直還是讀不懂古文，不知道怎麼拆解和斷句。
    不過很意外地在上了一個學期之後發現我居然看得種上課例句了。

    • 成績：A+

    • 課後時間：2 小時 / 週

    評分方式

    課堂小考 25% 、作業 15% 、期中考 30% 、期末考 30%

    上課方式

    每週會有課本單字小考，通常每一次是考半課單字，但後半學期課比較趕，所以有時候會一次考一課。
    作業是課本內每課的習題，基本上不難。

    師大的韓文聽說進度比台大快，而且一週只有兩小時。
    這學期上了高麗大學韓國語 1 的前六課。

    話說這個教授的中文超好，好到我一開始沒有發現是韓國人。
    因為目前的韓文文法幾乎和日文一樣，所以關於教授教得好不好我也不好評價，不過我覺得上她的課蠻開心的。
    比較不喜歡的一點是這堂課花了三週的時間在教 37 音，讓後面的課程有點趕，所以我後半學期單字背的不是很熟。

    期中期末

    大致上和作業題差不多，沒有聽力題，全部是韓文的問答題。

    感想

    大一就說要讀韓文了，結果到大三才開始讀。
    會修這門課主要是因為我想要強迫自己讀韓文，不然等到大學畢業之後應該就更沒有動力讀了。
    但我又選不到台大的韓文課，所以只好來師大了。

    學韓文的心得的話，因為文法和日文差不多，基本上我都是先用日文造出句子，然後把單字換成韓文。
    所以反而單字才是最大難關，後半學期遇到 ADA 改考卷和互動裝置期末 project，加上教授開始每週趕一課，導致我幾乎沒有時間背單字，每週小考都是靠考前抱佛腳，也因此一考完就全忘光了。

    不過希望下學期可以更認真學。

    其他修課以外的

    演算法設計與分析 - 呂學一教授（助教）

    • 課後時間：每次改考卷大約 30~40 小時，每週 1 小時 TA hour

    工作內容

    每週一小時的 TA hour。
    我自己 TA hour 之前是會先搞懂上課簡報，但另外兩個人我就不知道了。

    改考卷之前要先寫一份詳解給教授，教授改完之後再修改，然後再給教授改一次。
    來來回回直到教授覺得 OK 之後再開始改考卷。
    所以通常是禮拜四考完，五六日寫詳解 + 來回修改，大概下禮拜一才會開始改，兩週內改完比預想得還要趕。
    喔不過我還是要澄清一下我都有在兩週內改完。

    感想

    一開始會想當助教除了賺錢之外，也希望身為系邊的我可以和教授還有學弟們變熟。
    但現在我覺得我沒有被學弟們暗殺就該偷笑了。
    當助教還有另一個原因是因為我去年修 ADA 覺得很有趣，想知道理論演算法的世界適不適合自己。
    但在寫詳解的過程中讓我知道我只是喜歡看別人的神奇解法，自己對極度嚴謹的演算法本身沒有興趣。

    但當完一個學期的助教之後，確實讓我有從另一個視角參與一門課的體驗，可以看到當初修課時的現象背後存在的意義和理由，我還是覺得是個很新奇的體驗。
    在當完之後，可以理解教授對於課程的理念以及想傳達的事情。
    教授覺得在沒有平時作業的情況下，段考是唯一一個讓學生發現自己想法上的漏洞或是不嚴謹的機會，所以會希望助教可以指出所有的瑕疵，和一些學生們沒有考量完善的地方，所以才會變成大家看到的「助教改很嚴」的狀況。
    不過也確實在三次改考卷的過程中慢慢了解了教授量尺的界線了。

    另外教授本人比我原先想像的好，有時候他的回信禮貌到反而讓我有點不好意思。
    有時候我也在想教授會不會覺得我寫詳解的時候搞錯一堆，明明拿錢還做不好之類的，但後來看起來好像沒事，感謝教授沒有電我。

    這學期最大的失算大概就是接了兩個助教份量的工作吧。
    因為第二次段考離期末有點時間、第三次段考在期末週之後，原本預計這兩次應該不會太忙，但很不巧的互機裝置期末往前、沉浸式劇場期末往後，完美的撞上了改考卷的時間，結果害自己變超忙。
    話而去年助教只有兩個人還要處理一百多人的考卷，好像可以猜到為什麼助教會消失了。

    總結來說，我其實並不後悔當助教，一開始當助教的原因大部分也都達成了，也確實和教授變熟了不少。
