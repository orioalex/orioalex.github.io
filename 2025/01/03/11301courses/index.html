    修課心得
    台大, 心得

    這學期對我來說算是充滿意外的一學期吧，以為很重的課反而很輕，以為很難的課反而簡單，以為輕鬆的課結果爆炸，結果和當初學期初的預想完全不一樣。
    另外看到大一的學弟讓我有一些感慨，在和他們互動中發現自己這一年來似乎也有不少成長，意識到過去無心的一些選擇確實一定程度上改變了自己。

    以下是慣例的定義。

    甜度：認真修課的請況下，獲得分數的高低。

    1：有及格就不錯了
    2：不至於被當，但分數不好看
    3：分數中規中矩，A+ 不太可能
    4：就算沒有 A+ 也不會拿到太差的分數
    5：認真的話 A+ 沒什麼問題

    課後時間：課後需要的時間。

    收獲：在這堂課學到的知識量。

    1：基本上沒有
    2：雖然有但不多
    3：有一些知識
    4：有一定程度的知識量
    5：滿滿的收獲，物超所值

    綜合推薦度：是否會推薦別人來修這堂課。

    1：逃げろ!
    2：非修不可再修
    3：如果缺課還是可以
    4：時間允許建議修
    5：有時間必修

    必修課程

    線性代數 - 李明穗教授（3 學分）

    • 分類：必修
    • 成績：A+
    • 甜度：4.5
    • 收獲：3
    • 推薦度：4
    • 課後時間：1 小時 / 週

    評分方式

    作業 30%、兩次期中考各 20%、期末考 30%。
    有 10 份作業，去掉一次最低分，剩下取平均。

    教授第二和第三次段考的時候有發糖果，大推。

    上課方式

    剛開學的時候有一份加簽用作業，不算分，但不管是要加簽的還是已經選上的人都必須要寫。
    教授進度上得蠻慢的，花了不少時間在我覺得不需要太過著墨的地方，不過這對我而言也不錯就是了，我也不想花太多心力在數學上，隔壁班的線代看起來好難。

    還有教授有時候上課上到一半會開始聊一些她的留學回憶，像是一些學校間的名爭暗鬥之類的。

    這堂課確實如傳言一般 loading 很輕，輕到我覺得這堂課比較像是「矩陣鑑賞與評析」，整學期大致上就是看了各式各樣的矩陣，介紹它們的性質，然後就差不多了。
    我自己覺得有趣的一點是線性代數用了更數學的方法去定義向量和向量運算，所以除了高中學過的向量以外，矩陣也可以是向量，多項式也可以是向量，或是一些奇怪的東西也是向量。

    然後這個教授課後不會提供影片、也沒有上課筆記，只有一些定理和證明，所以筆記要自己抄。
    但我很懶再加上進度不快，所以我就索性不抄了，反正段考考題不難，作業寫完就差不多會了。

    期中期末

    我自己覺得算簡單，體感上和作業差不多難，除了一些陷阱題要注意以外就沒什麼問題了。
    不過我第一次段考大失誤，搞錯題目意思導致一個大題直接零分，當時甚至在想要不要停休，不過還好後面兩次段考把自己救回來了。

    感想

    雖然課堂上有提到一些特徵值等等的應用，但總覺得自己還不是很清楚具體來說要做什麼。
    或許未來修影像處理的時候就會知道了吧。

    演算法設計與分析 - 呂學一教授（3 學分）

    • 分類：必修
    • 成績：
    • 甜度：
    • 收獲：4.5
    • 推薦度：
    • 課後時間：三次段考共約 10 小時

    評分方式

    沒有公告評分標準。
    但聽說是三次段考取平均，再視情況加一些調分。

    上課方式

    這門課沒有任何的作業，所以除了段考以外就沒有任何的 loading 了。
    這門課另外一個特色大概就是不用寫程式吧，對於所有的演算法，只要能夠口述說明就可以了。
    不過相對的，這堂課會要求證明演算法的正確性。

    針對每一個主題，教授會先給出一個問題，然後開始講比較早期的演算法，然後慢慢講到接近現代的演算法，過程中也當然沒有任何的程式，而只有邏輯說明和正確性/複雜度的證明。
    我自己覺得上他的課有點像是看魔術表演一樣，「用各種神奇的方式達成看似不可能的任務」，對我來說，每一次的上課都會讓我期待還有什麼新招去破解難題。
    另外我覺得先講早期解法再推到現在的解法，比起一開始就講最終解法要來得有趣，再加上教授自己是做演算法的，所以會知道一些奇聞軼事和歷史流變，我自己感覺比較像在聽故事一樣，相比於單純地介紹演算法來得更有趣。
    不過如果是喜歡實務應用層面的話，隔壁黃上恩班應該會更好。

    段考

    這門課有 3 次段考，不過因為第一和第二次段考間放了 4 次假，所以後來決定第二次段考範圍包括第一次的內容。
    大概是因為今年沒有大抄的關系，題目的量相對變得比較少，我寫完之後大概還有 15 分鐘來把自己的答案檢查過一次。
    題目比例似乎也不一樣了，課內證明題大約佔一半再少一些些，剩下的就是課內的延伸題。
    感覺就算沒有寫得很嚴謹，只要邏輯對就會幾乎給滿分，但也可能只是因為我的考卷是助教改的而已。

    考卷不會剛好只有 100 分，通常會有一到兩題的失誤容許度，不過前兩次段考的 quota 都被我拿去給 DP 用了。
    然後這堂課是不會公佈成績比例的，你只會知道你自己的成績。

    感想

    雖然很多人不太喜歡這堂課，不過我自己倒是蠻慶幸自己在這裡而不是在隔壁班。
    原本預計自己的成績會像 DSA 一樣不理想，但結果似乎還不錯。
    一個最主要的原因大概是我不太擅長寫程式，呂學一班這種口述邏輯的方式就不用去處理那些麻煩的 bug 和邊界特判，對我來說反而輕鬆很多。比起 DSA 那種超難的程式作業，理解演算法思路對我來說要來得簡單多了。
    這門課也讓我知道演算法研究的世界原來是不寫程式，而是由邏輯和證明堆積起來的。

    另外在最後的 P=NP 問題上也和自動機剛好可以互補，在自動機中學到了 P=NP 問題的原始定義，而在 ADA 中了解到了這個問題在演算法層面上的意義，雖然先前就有聽過這個問題，但是是直到修了這兩門課才讓我覺得自己真正了解了 P=NP 問題，也有一種自己終於有能力去碰觸這個末解之謎的感動。

    系統程式設計 - 黎士瑋教授（3 學分）

    • 分類：必修
    • 成績：
    • 甜度：
    • 收獲：4
    • 推薦度：
    • 課後時間：
      • 期中期末：5 小時 * 2 次
      • 程式作業：四次作業分別約 20、30、5、10 小時
      • 手寫作業：約 1 小時 * 4 次

    評分方式

    程式作業 4 次共 32%、手寫作業 4 次共 8%、期中期末考各 30%。

    上課方式

    整學期的語法課居多，會旁及一些相關的概念。
    大致上是在介紹你在寫一個系統的時候會用到的概念和 C 語法，像是 communication、lock、fork、thread 等等。
    我自己覺得這堂課算有趣，可以讓自己了解到原來當初計程學到的 C 語言還遠遠不是它的全貌，修了這門課才發現它能做的事比我當初預想的要多許多。

    程式作業

    程式作業應該是這一門課最花時間的地方，或甚至是整學期的主要 loading。
    一次作業的 deadline 大約是 2 週，除非碰到段考週等等狀況才會改 3 週。
    我這學期 loading 基本上就是隨著 SP 作業跑，每次就是 SP 作業一出 loading 就瞬間變重，作業一寫完 loading 就瞬間不見，跟坐雲宵飛車一樣。

    因為自己寫 SP 作業時遇到一堆奇怪問題，所以寫成一篇 SP 靈異事件表。

    期中期末

    必須說，雖然我覺得這堂課很有趣，但我自己不太喜歡它的段考。
    我的思想迴路似乎和助教們不太一樣，明明考出來的東西我自認我都知道，但總是因為作答不充分或是沒有寫到關鍵字等等原因被扣分。

    像是我寫「在第 X 行使用 pthread_join 來解決問題」，但因為沒有寫 pthread_join 的參數被扣分。
    又或是題目問 A 和 B 的差別，然後我只寫了一個裡由，但助教說要寫二個以上才全給分之類的。
    又或者我寫說「因為 passwd 是 set uid process 所以 user 執行它之後可以獲得 root 權限修改 /etc/passwd」，但因為沒有講到 /etc/passwd 對 root 有 write 權限以及 passwd 對一般人有 execute 權限所以披扣分。

    居然是因為這些原因害我段考被扣了十幾分，讓我自己不是很開心。
    另外一個讓我段考考糟的原因是我自己很不擅長手寫 code，這點從 DSA 的時候就很明顯了，自己總是會漏判一些 case 導致自己寫出來的 code 充滿奇怪的 bug，不過這個就完完全全是我的問題了，只能說我真的和 code 處不來。

    感想

    之前看了別人的修課心得，感覺這是一堂輕鬆的課，實際修的時候確實也覺得蠻輕鬆的，直到段考成績出來以前，我都以為自己可以輕鬆 A+ 掉這門課，看到線代二段有 97 的時候還覺得自己這學期搞不好可以 4.3，結果看到段考 79 分直接把我打回原型。

    不過單論課程內容的話，我還是覺得學到一些系統相關的邏輯和架構是一件蠻有趣的事。
    另外，修了這門課之後我才知道原來 C 連作業系統都可以寫，讓我開始好奇下學期的 OS 了。

    自動機與形式語言 - 林智仁教授（3 學分）

    • 分類：必修（大三）
    • 成績：A+
    • 甜度：4
    • 收獲：4
    • 推薦度：4
    • 課後時間：2.5 小時 / 週

    我們用的課本叫做「Sipser_Introduction.to.the.Theory.of.Computation.3E」所以比起自動機，這門課應該是計算導論才對。
    不過確實這門課花了大部分的篇幅在自動機和形式語言上，所以好像也沒有錯。

    評分方式

    作業 6 次共 10%、兩次期中考和一次期末考各 30%，每年不太一樣。
    蠻佩服的一個是，這班的段考考卷是教授自己親自改的，一個 100 多人的班居然有辦法一個禮拜就改完。
    等第是依照隔壁林忠緯教授的班切，想要知道成績比例的話可以看教授的個人網頁，改網址的話就可以看到前幾年的成績。

    上課方式

    一開始我有猶豫過要加簽哪一個班，後來決定林智仁班主要是因為不用出席、而且有作業分，再加上這個班不限外系加簽，在固定比例的情況下感覺得分會比較簡單。
    教授上課時會播放他事前就預錄好，已經放在教授個人網站上的影片，雖然上課時教授人也在場，但基本上只是播影片而已，偶爾會做一些補充說明，但我個人覺得那些補充說明不是很重要。

    作業大部分都是課本題、之前的段考題、再不然就是 google 一下找得到答案的題目，我自己覺得算好拿分。
    另外影片的語速超級慢，對於我這種國榮微積分都開兩倍速的人來說實在是太慢了，所以後來我也不太愛看影片而是改看原文書了。

    開學到第一次段考前的期間，我為了避免自己漏聽教授的補充，所以就還是都會到教室，然後坐在座位上看原文書。
    原文書的內容意外地寫得很好懂，我後來就只看原文書而沒有再去看教授的影片和 PPT 了，上課時間也被我拿去寫歌而沒有去教室了。
    不過也因為如此害我不知道第二次段考跳過了大部分的 DPDA，害我想說怎麼突然進度那麼快，差點讀不完段考範圍。
    第二到三次段考的期間，因為章節跳的比較多，再加上有上次段考的前車之鑑，所以我就大致刷過教授的影片，然後在原文書裡找對應的部分讀了。
    不過教授似乎沒有把課程網頁上的影片全部播完，所以我猜我還是多讀了一些東西就是了。

    感想

    這門課真的如傳言所說，不需要任何先備知識，甚至我覺得它比大一計程更適合大一修。
    有趣的一點是，這門課一開始會花很多時間在一些看似意義不明的東西，直到最後才會知道原來這些都是為了進入圖靈機的準備。
    不過有一點可惜的是，這門課花了大約三分之二個學期在構造圖靈機和它的先備知識，反而正要進入正題的時候就結束了。畢竟這門課的精髓應該是在探討圖靈機（電腦）的能力，結果反而跳了不少圖靈機後的章節，個人覺得有點可惜。

    一開始讓我發現這門課的用處其實是在申請交換學生的時候，那時候在思考要如何把資訊和語言結合，結果意外地找到「計算語言學」這個領域。發現自動機課堂上講到的 CFG 似乎就是用作語言處理的一個手段，意外地讓我發現自動機也不一定只有計算理論。

    另外加簽的時候發生了個小插曲，一開始我兩個班的自動機都有去簽，結果以旁聽生身份被加入林忠緯班、以正式選課生被加入林智仁班，所以就變成我會收到兩個班的 NTU cool 公告，再加上兩位教授都不愛在公告署名，導致我有時候讀完才發現根本是另一個班的公告，所以後來我每次收到公告都還要去 cool 上面確認到底是哪個班的。

    現在看什麼都是 Halting Problem

    選修課程

    • 分類：服務學習
    • 成績：N/A
    • 甜度：N/A
    • 收獲：2
    • 推薦度：N/A
      • 不考慮三階的話是 5
      • 但我也不知道三階是好是壞
    • 課後時間：10 小時 / 學期

    法文二上 - 葉麗儀教授（3 學分）

    學好法文的關鍵不是和法國人講話，而是講法文。

    • 分類：一般選修（因為外語滿了）
    • 成績：A+

    評分方式

    平時成績 20%、小考 20%、期中考 30%、期末考 30%，加分一次 +2、上限 +10。

    平時成績應該是看出席，所以全勤的話應該就是滿分，如果這部分有被扣分的話可以透過加分補回來，補在平時成績的加分不計入加分上限。

    這學期的小考考的是 l’imparfait 和 le futur simple 兩個時態變化，只考上課講過的單字，所以有背的話沒什麼問題。

    加分的話，這學期有背聽力文稿（兩篇）、和法國人交流（兩次）、寫自己的愛情史，共 5 次加分機會。
    我自己只拿了兩次就是了，不過我覺得這也是有心就能拿滿的分數。

    期中考平均 90.63，期末考平均 73.63，算是這堂課唯二比較容易失分的地方吧。

    選課方式

    這堂課的選課比較特別，需要先考過課前考才可以選課 / 加簽，沒過的人就算有選上也會被強制退選。
    課前考的內容就是動詞變化，以前會考現在式(indicatif présent)、複合過去式(indicatif passé composé)、簡單過去式(indicatif imparfait)、簡單末來式(indicatif futur simple)，不過今年只有考前兩個而已。

    考的方式就是給出動詞和主詞，寫出現在式和複合過去式。
    考試範圍是市面上隨便一本動詞變化字典，不過我是用這個單字網站配上動詞變化練習機來練。
    （單字網站出處）。但是那個單字網站好像有一些單字有寫錯，不過在動詞變化練習機練的時候應該就會發現了，所以我覺得問題不大。
    只要 60 分就過了，對於有背的人應該不是很難，我自己是趁暑假的時候就把它背完了。

    然後課前考我還背錯 dire。

    上課方式

    先前就有耳聞說這個教授很在乎連音，不過這學期感覺下來，比起「在乎連音」，應該是「在乎口說」這個說法比較準確。
    上這個教授的課的時候可以感覺得出來她的發音真的很好，我猜教授應該是有下過功夫在口語上，我覺得能學到標準的口說對我而言反而是一件好事，再加上我也認同要弄懂發音後才能拆解聽到的文字這件事，所以我反而是同意教授強調連音這件事。
    我也是上這學期的課才知道原來很多音我原本都發錯了，也順便讓我發現日文的清濁音根本就不是差在送氣的問題。

    另外，之前在查評價的時候查到有人說這堂課很紮實，上完之後我自己感覺確實如此，不過我一開始以為這個紮實是進度飆很快的意思。後來才知道，紮實指的是對於一個文法的訓練量很大，當我們學了一個新的文法的時候，教授會讓我們看文章，去體會那個文法的語境，然後會設計一個情境要我們互相對談，下次上課的時候會和學生問答來練習文法。所以紮實指的不是內容多，而是指訓練量大。
    也因為課內會有很多互動的關係，對於像我這樣的社恐來說有點壓力，有時候分組遇到大家都是社恐的時候就會整組很安靜。
    不過畢竟對話也是學習語言的一環，所以還算可以接受就是了。

    教授以前好像是在法國學語言學的，她對於文法結構非常了解，所以學到一個新文法的時候，她會把其他相似的文法也一起講。
    我們這學期就體驗了代名詞全餐。

    前半學期講了過去和末來式，後半則講了各種代名詞和祈使句，還有代名詞+祈使句、代名詞+過去式等等的文法炸彈 ，然後就會發現不管什麼文法只要跟過去式參上一腳就會變得很可怕。

    感想

    教授人蠻好的，有空的時候她會常常和學生聊天，不過我們這節課前後她都有事，沒有太多可以講話的機會，有點可惜。
    另外感覺她不希望給學生太多學法文的壓力，有人問答答不出來的時候，她會說「啊上次我這邊好像沒有講清楚」等等，情緒價值給好給滿，認真聽的話其實可以從她身上學到不少說話技巧。

    她上課的時候也常常會分享一些法國留學生活和文化衝擊，她總是有辦法講得很有畫面感。
    像是她之前分享說法國的跨年沒有倒數，所以大家都自己數自己的，然後每個人跨年的時間點都不一樣等等，這件事真的讓我印象深刻。

    沒意外的話下學期應該也會繼續修她的課。

    要注意的是，期中報告的截止時間好像是晚上 10 點，而且遲交零分，要壓死線的人要小心。

    期末

    期末算一算我只要達到 30 分的條件就可以，所以就輕鬆很多。
    期末考得蠻難的，還是需要花時間背簡報，不過因為都是是非和選擇題，沒記錯的話期望值就有 30 分了，所以大略背一下就好。
    最後我是拿了 45 分。

    感想

    對於課程本身沒什麼特別的感想，不過有一段探討「人類為什麼會想要戰爭」的段落，裡面講到戰爭是一個很輕易能夠獲得榮耀和成就的地方，所以總是會有人民響應戰爭，這個觀點確實蠻有趣的。

    加簽方式

    加簽是按照日文系 > 外系大四 > 外系大三 > … 的順序下去簽，但今年只簽了一半的外系大四，基本上大四以下的大概是沒有機會簽到。
    但是這堂課開放旁聽，而且旁聽生也可以參加段考，加入 cool 等等，除了學分以外和一般生沒什麼差異，我自己大概聽了四分之三個學期吧。

    上課方式

    一開始會收 160 元的課本費用，課本裡面就是各種情況的書信往來範例。
    學期一開始會從「請、謝謝、對不起」開始，然後按照格式構造出一個段落，再把段落拼成一封書信。

    我自己覺得即使是旁聽生也可以學到不少東西，在這學期之後我確實可以感覺到自己比較知道日文的書信要怎麼寫了，這也反過來對我的中文書信有一些幫助。
    這堂課我覺得有趣的一點是，上完之後才知道，不是把動詞變成特別尊敬語就會很禮貌，很多時候還需要透過一些說話的藝術來潤飾文句來讓收信者感到舒適。

    還有學完之後你就可以用超長的敬語來煩別人。

    期中期末

    期中第一大題是考書信位置和結構，像是收信人寫在哪裡，請、謝謝、對不起要寫在第幾段，信件結尾要寫什麼等等。
    第二大題是詞語解釋，用中文或日文解釋題目中的日文單字，單字應該都是從課文內出。
    第三大題是按照上課中的格式寫出段落。
    第四大題是請、謝謝、對不起的造句。
    第五大題是按照情境寫一封書信。

    期末我沒有去考，所以不知道。

    お忙しいところ、わざわざ貴重なお時間を割いていただきまして、この長い文章を読んでくださり、誠に有難く心から感謝致したく存じます。

    社團

    數位音樂創作社

    這學期開始當了社長，不過感覺自己最有收獲的地方是在教社課的時候，當初為了不要被問題考倒，所以有多翻一些資料來確保自己真的把要上的東西完全弄懂了，雖然到最後都沒有人提出問題就是了。不過自己很清楚的是，作為一個講者，反而比起一年前在台下聽的自己學到得要來得多很多。

    總結

    心得

    自己在這一年半裡不知不覺間學了不少東西，已經不再是剛進大學時那樣對資訊一竅不通的人了，在聽新聞媒體報導的時候會意識到有些東西並不僅僅是大家表面上看到的那樣，感覺自己在資訊方面已經不是一個懵懂無知的外行人了。
    在跟學弟們的互動中，有時候會讓自己意識到自己的成長，像是被問到一些自己大一時也搞不懂的問題的時候，發現一年後的自己已經能夠回答這些問題。或是自己回去寫 judgegirl 的時候發現自己可以輕鬆地把邏輯轉變成程式碼等等。
    雖然我總是說自己的課太涼等等的，但很多事情都向自己證明了過去的學習對自己而言是真實存在且有意義的。

    希望明年的自己也可以有新的體悟。

    回顧

    因為上面講到的原因，突然想看看當初大一修的課對大二的自己而言留下了什麼。

    計算機程式設計

    我第一個蹺課的課，算是教會我蹺課的必要性吧。

    計算機概論

    讓我理解了電腦裡面的各個部件到底是在做什麼，還有一個語言是怎麼被翻成機器語言的。
    現實日常中的實用性應該只有挑電腦的時候用過而已，但知識是確實有留下來的。

    法文一上下

    沒修法文一的話現在就不會修法文二了，而且我確實是喜歡法文的。

    永續發展與社會 (一)

    意外地有用。
    我在這堂課學到了環境議題的各個層面，有時候看到網路上對核能議題的評論，就會覺得很多人在評論之前似乎完全沒有去了解一個議題，而只是靠著自己的想像去議論而已，也讓自己知道缺乏背景知識的議論完全沒有實質意義。

    資料結構與演算法

    至少自己確實學到了像是 skip list、treap 等等以前不會的東西，而且確實有些在 ADA 的時候又用到了。

    網路管理與系統管理 (NASA)

    最有用的應該是 shell script。
    在 DSA 的 debugger、SP 的 judge，會寫 shell script 的話確實可以幫自己解決一些自動化的問題，甚至可以說是全部課程裡面使用率最高的技能。

    目前第二有用的應該是 LDAP，但我好像只是拿來查同學的個資而已。
    NFS 的話以後如果買了 NAS 的話應該用得上吧。
    nginx 以後如果要架網站可能會有用？

    以 Excel 進行資料分析

    超有用。
    原本只是想水個通識學分，沒想到意外地有用。
    很多需要處理 excel 檔的時候就會慶幸自己有上過這堂課。
    像是之前在準備法文課前考的時候，就自己用 excel 寫了一個動詞變化機來用。
